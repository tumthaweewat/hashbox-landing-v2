## 3. Publishing Cadence

**ความจริงสำหรับทีมเล็ก:** อย่าสัญญา weekly ถ้าทำไม่ไหว — cluster ที่ค้างครึ่งทางแย่กว่าทำช้าแต่ครบ

**Cadence แนะนำ: 1 บทความ / 2 สัปดาห์** (เสร็จทั้ง cluster ใน ~10 สัปดาห์)

| สัปดาห์ | งาน |
|---|---|
| 1 | ปรับ Hub ให้เป็น answer-first + ใส่ schema + TOC placeholder |
| 1–2 | **เผยแพร่ S1 (GEO vs SEO)** |
| 3–4 | เผยแพร่ S3 (Checklist) + อัปเดตลิงก์ใน hub |
| 5–6 | เผยแพร่ S5 (Schema) + ลิงก์จาก `/schema-markup-thai-guide-2026/` |
| 7–8 | เผยแพร่ S2 (Get Cited) |
| 9–10 | เผยแพร่ S4 (Measuring) + ปิด cluster ครบ + ใส่ internal link ครบทุกทิศ |

**ทุกครั้งที่ publish spoke ใหม่ → กลับไปแก้ hub และ spoke ที่เกี่ยวข้องให้ลิงก์หากันทันที** (ห้ามค้าง)

### บทความที่ควร publish ก่อน: **S1 — GEO กับ SEO ต่างกันอย่างไร**

**เหตุผล:**
1. **ดึง intent ของ "geo คือ" ได้ตรงที่สุด** — คนที่ค้น "geo คือ" ส่วนใหญ่รู้จัก SEO อยู่แล้ว การเปรียบเทียบคือสิ่งที่เขาต้องการ จึงเสริม keyword หลักที่มี 156 impressions โดยตรง
2. **เขียนง่ายและเร็วที่สุด** — เป็น comparison ใช้ความรู้ที่ทีมมีอยู่แล้ว ไม่ต้อง research หนักเหมือน S4/S5 → ได้ momentum เร็ว
3. **เป็น "ประตู" เข้า cluster** — comparison content ติด featured snippet ง่าย และส่งคนต่อไปยัง checklist/how-to ได้ธรรมชาติ
4. มี keyword เดิมที่เกี่ยวข้องรองรับ (`technical seo คือ` อันดับ 27) ให้ลิงก์โยงกันได้ทันที

## 4. Template โครงสร้างเพื่อให้ AI Citation (ใช้ซ้ำได้ทุกบทความ)

AI engines (ChatGPT, Perplexity, Google AI Overviews) มักหยิบ content ที่ "ตอบคำถามได้ทันที, มีโครงสร้างชัด, ตรวจสอบได้" ไป cite ใช้โครงนี้กับทุก spoke:

```
┌─────────────────────────────────────────────────────────┐
│ 1. H1 = คำถามหรือ keyword จริง                            │
│    เช่น "GEO คืออะไร?" ไม่ใช่ "มารู้จัก GEO กันเถอะ"        │
├─────────────────────────────────────────────────────────┤
│ 2. ANSWER-FIRST (2–4 ประโยคแรก)                           │
│    ตอบคำถามให้จบใน 50–60 คำ ก่อนเล่ารายละเอียด            │
│    → นี่คือส่วนที่ AI หยิบไป cite บ่อยที่สุด                 │
├─────────────────────────────────────────────────────────┤
│ 3. DEFINITION box                                         │
│    นิยามชัด ๆ 1 ประโยค รูปแบบ "X คือ ..." (bold คำหลัก)    │
├─────────────────────────────────────────────────────────┤
│ 4. เนื้อหาแบ่งด้วย H2/H3 ที่เป็นคำถาม                      │
│    ("ทำไม...", "อย่างไร...", "ต่างกันอย่างไร...")          │
├─────────────────────────────────────────────────────────┤
│ 5. STATS / ตัวเลข อย่างน้อย 2–3 จุด                        │
│    มีตัวเลข + แหล่งอ้างอิง → AI ชอบ cite ข้อมูลที่มีตัวเลข   │
├─────────────────────────────────────────────────────────┤
│ 6. LISTS & TABLES                                         │
│    bullet / numbered / ตารางเปรียบเทียบ — parse ง่าย       │
├─────────────────────────────────────────────────────────┤
│ 7. FAQ section (4–6 ข้อ) + FAQPage schema                 │
│    แต่ละคำถามตอบสั้น ๆ จบในตัว → ติด People Also Ask + AI  │
├─────────────────────────────────────────────────────────┤
│ 8. AUTHOR + วันที่อัปเดต + แหล่งอ้างอิง                    │
│    ใส่ชื่อผู้เขียน (Tum Thaweewat) + Person/Article schema  │
│    → E-E-A-T signal ที่ AI ใช้ตัดสิน trust                 │
└─────────────────────────────────────────────────────────┘
```

**Checklist ก่อนกด publish ทุกบทความ:**
- [ ] ย่อหน้าแรกตอบคำถามจบใน 60 คำ (answer-first)
- [ ] มี definition แบบ "X คือ ..." อย่างน้อย 1 จุด
- [ ] มีตัวเลข/สถิติพร้อมแหล่งอ้างอิง ≥ 2 จุด
- [ ] มีตารางหรือ list อย่างน้อย 1
- [ ] มี FAQ ≥ 4 ข้อ + FAQPage schema
- [ ] ใส่ Article + Person schema (ชื่อผู้เขียน)
- [ ] internal link ออก ≥ 3 (hub + 2 spoke/service) ด้วย keyword anchor
- [ ] H2/H3 เขียนเป็นคำถามที่คนค้นจริง
